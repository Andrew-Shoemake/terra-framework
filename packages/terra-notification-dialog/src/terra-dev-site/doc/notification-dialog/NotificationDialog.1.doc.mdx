# Terra Notification Dialog

The `terra-notification-dialog` component renders as a notification using the `terra-abstract-modal` to display a dialog over content. It is a common component that can be used to display a message to a user. When the user is presented with the dialog, it is paired with an acceptance and an optional rejection action to allow the user to correctly handle the notification.

The dialog will have the highest z-index of 9000, so it will display over all other content, including over modals.

There are five dialog variants built into `terra-notification-dialog`:

1. `alert`
2. `error`
3. `warning`
4. `info`

If one of these variants do not meet your needs, there is the option to customize the signal word and the icon that the dialog displays.

By choosing to create a custom Notification Dialog, you are responsible for providing an icon that is themed correctly for the terra-default-theme, orion-fusion-theme, and clinical-lowlight-theme. See Terra's [Theme Strategy Guide](/guides/terra-application/theme-strategy) for more info.

## Getting Started

- Install with [npmjs](https://www.npmjs.com):
  - `npm install terra-notification-dialog`

## Usage 
``` jsx

```

## Content Layouts
The Notification Dialog provides the following layouts for the `content` prop:

### ContentLayoutAsList
The layout styles list of strings. Import like:
``` jsx
import { ContentLayoutAsList } from 'terra-notification-dialog';
```
